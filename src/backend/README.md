# Robert Backend (FastAPI)

## Description

This backend powers the Robert AI Medical Assistant.  
It provides chat, session, GDPR, and transcription endpoints, and uses a SNOMED-grounded RAG pipeline for EMR-aware responses.

## Files and directories

| Path | Purpose |
|---|---|
| `app/main.py` | FastAPI app entrypoint and startup resource loading. |
| `app/routers/` | API routes (`chat`, `sessions`, `gdpr_router`). |
| `app/services/` | LLM, RAG, privacy, session, and speech services. |
| `app/cli/` | CLI tools for CUI search, graph expansion, and EMR retrieval. |
| `data/` | Local EMR sample files and session data directory. |
| `tests/` | Pytest suite for services and endpoints. |
| `scripts/benchmark.py` | Benchmark script for retrieval/response modes. |
| `.env.example` | Environment variable template. |
| `pyproject.toml` | Project metadata and dependencies for `uv sync`. |
| `requirements.txt` | Python dependencies for `pip`-based setup. |
| `uv.lock` | Locked dependency graph generated by `uv lock`. |

## Prerequisites

- Python `3.12+`
- `uv` **or** `pip` + `venv`
- Internet on first run (model downloads)
- `GEMINI_API_KEY`
- `SARVAM_API_KEY` (required for translation/TTS/STT features)

## Data files (in `src/`, one level above `backend/`)

| File | Required | Purpose | Link |
|---|---|---|
| `GraphModel_SNOMED_CUI_Embedding.pkl` | Yes | SapBERT CUI embedding index. | [Link](https://drive.google.com/file/d/1a2axTk35wsvQ4AJOheZnjZJdyksHg1cy/view)
| `SNOMED_CUI_MAJID_Graph_wSelf.pkl` | Yes | SNOMED knowledge graph. | [Link](https://drive.google.com/file/d/1zlb0zey_tAnFWtCY_NvhA0dqfydL4Ph7/view) 
| `sm_t047_cui_aui_eng.pkl` | No | CUI -> names mapping. | [Link](https://drive.google.com/file/d/1xnZyz_ePAcXzmzCaqJHsAI0sf8LsG8DA/view)

If required `.pkl` files are missing, startup fails while loading RAG resources.

## Installation and setup (uv)

### 1) Install `uv`

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
source "$HOME/.local/bin/env"
```

### 2) Install dependencies

```bash
cd src/backend
uv sync --extra dev
uv run python -m spacy download en_core_web_lg
```

## Installation and setup (pip + venv)

```bash
cd src/backend
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
python -m spacy download en_core_web_lg
```

## Configure environment variables

```bash
cp .env.example .env
```

Set values in `.env`:

| Variable | Required | Purpose |
|---|---|---|
| `GEMINI_API_KEY` | Yes | Gemini chat + compaction backend. |
| `SARVAM_API_KEY` | Yes (for multilingual/voice) | Required for translation, TTS, and `/transcribe` STT endpoint. |

If `SARVAM_API_KEY` is not set, Sarvam-powered features will fail.

## Ollama setup (MedGemma)

The backend supports Ollama via model name used in code:
`MedAIBase/MedGemma1.5:4b`

### macOS

```bash
brew install ollama
brew services start ollama
ollama pull MedAIBase/MedGemma1.5:4b
```

### Arch Linux

```bash
sudo pacman -S ollama
sudo systemctl enable --now ollama.service
ollama pull MedAIBase/MedGemma1.5:4b
```

Verify:

```bash
ollama list
```

## Running the backend

Using `uv`:

```bash
cd src/backend
uv run uvicorn app.main:app --host 0.0.0.0 --port 8013 --reload
```

Using `pip` virtual environment:

```bash
cd src/backend
source .venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8013 --reload
```

Backend base URL: `http://localhost:8013`

## API endpoints

| Method | Path | Description |
|---|---|---|
| `GET` | `/` | Root status payload |
| `GET` | `/health` | Health check |
| `POST` | `/chat` | One-shot chat endpoint |
| `GET` | `/sessions/{patient_id}` | List sessions for a patient |
| `POST` | `/sessions/{patient_id}` | Create session |
| `GET` | `/sessions/{session_id}/messages` | Get session messages |
| `POST` | `/sessions/{session_id}/message` | Send message in a session |
| `POST` | `/transcribe` | Sarvam speech-to-text |
| `POST` | `/gdpr/consent/{session_id}` | Update consent flags |
| `GET` | `/gdpr/evidence/{session_id}` | Evidence timeline (Art. 15) |
| `DELETE` | `/gdpr/sessions/{session_id}` | Delete one session (Art. 17) |
| `DELETE` | `/gdpr/patient/{patient_id}` | Delete all sessions for patient (Art. 17) |

## Useful CLI commands

`uv` style (default in this README):

```bash
cd src/backend

# CUI search
uv run python -m app.cli.cui_search "my head hurts" --show-terms

# Graph expansion
uv run python -m app.cli.graph_expand "chest pain" --depth 1 --max-per-hop 20

# Full retrieval pipeline
uv run python -m app.cli.emr_retrieve "my head hurts" --emr data/patient101.json --show-prompt

# Benchmark
uv run python -m scripts.benchmark \
  --emr-path data/patient101.json \
  --embedding-path ../GraphModel_SNOMED_CUI_Embedding.pkl \
  --graph-path ../SNOMED_CUI_MAJID_Graph_wSelf.pkl
```

If you use `pip` + virtualenv, run the same commands with `python -m ...` instead of `uv run python -m ...`.

### macOS note for RAG commands (OpenMP/MKL)

On some macOS setups, Torch/FAISS/BLAS combinations can trigger OpenMP runtime conflicts or excessive thread contention while running RAG CLIs.

If you see OpenMP-related issues (for example `OMP: Error #15`) when running `graph_expand`, `emr_retrieve`, or `benchmark`, prefix those commands with:

```bash
KMP_DUPLICATE_LIB_OK=TRUE OMP_NUM_THREADS=1 MKL_NUM_THREADS=1
```

Example:

```bash
cd src/backend
KMP_DUPLICATE_LIB_OK=TRUE OMP_NUM_THREADS=1 MKL_NUM_THREADS=1 \
uv run python -m app.cli.emr_retrieve "my head hurts" --emr data/patient101.json --show-prompt
```
